# Макет: страница «Мой аккаунт»

> Цель: убрать «пустоту» после удаления тестовых кнопок уведомлений, не дублируя будущий раздел «Статистика».

## Текущее состояние

```
┌─────────────────────────────┐
│  ←  Мой аккаунт             │
├─────────────────────────────┤
│         (аватар)            │
│         Имя                 │
│  [Мгновенное уведомление]   │  ← убрать после тестов
│  [Запланированное уведомл.] │  ← убрать после тестов
│         email / Гость       │
│                             │
│  (для гостя: баннер + форма │
│   регистрации)              │
│                             │
├─────────────────────────────┤
│      [ Выйти ]              │
└─────────────────────────────┘
```

**Проблема:** у зарегистрированного пользователя остаётся только шапка + «Выйти» — страница выглядит незавершённой и не даёт ценности.

---

## Принцип: «обзор + данные аккаунта», не «статистика»

| Раздел «Статистика» (вкладка, в разработке) | Страница «Аккаунт» |
|---|---|
| Графики, история приёмов, тренды | Краткий снимок «сейчас» |
| Детальная аналитика по периодам | 2–3 цифры без графиков |
| Основной сценарий просмотра | Кто я + данные профиля + краткий обзор |

Мини-цифры на аккаунте — **тизер**, не замена вкладке «Статистика».

Быстрые переходы (профиль, шаринг, подопечные) **не нужны** — они уже доступны с вкладки «Профиль» и меню «Добавить».

### Смена пароля: когда показывать

| Способ входа | Пункт «Сменить пароль» |
|---|---|
| Только Google (`google.com`) | **Скрыт** |
| Email + пароль (`password`) | **Показан** |
| Google + привязанный пароль (оба провайдера) | **Показан** |

Проверка на стороне Firebase: `user.providerData` содержит запись с `providerId == 'password'`.  
Аналог уже есть в `EmailAuthProvidersLookup.hasPasswordProvider` (`lib/core/auth_email_lookup.dart`).

Для Google-only в шапке достаточно email/имени — отдельно указывать способ входа не обязательно.

Для гостя **не добавляем**: мини-обзор, блок «Данные профиля», «О приложении» (по желанию версию можно оставить внизу — опционально).

## Вариант для подопечного (`isWard`)

Упрощённый экран:

- шапка-профиль
- мини-обзор только по **своему** профилю (лекарства / сегодня к приёму)
- блок «О приложении»
- «Выйти»

Без: «Данные профиля», смена пароля (аккаунт создаёт опекун).

## Мини-обзор: что показывать

Только для **зарегистрированного** владельца аккаунта (не гость, не подопечный с ограничениями — для ward см. выше).

| Плитка | Источник данных | Зачем |
|---|---|---|
| **Всего лекарств** | `medicationNotifierProvider(currentUserId)` → длина списка | Все добавленные курсы, включая завершённые |
| **Активных** | `isMedicationCourseActive(med, now)` из `course_schedule.dart` | Курсы, у которых срок приёма ещё не истёк |
| **Подопечных** | `linkedUsersProvider` → `linkType == ward` | Напоминает о семейном сценарии |
| **Сегодня к приёму** | сумма `todaysIntakes` по активным курсам | Полезно каждый день, не дублирует графики |

**Не показывать здесь:** процент соблюдения, недельные графики, историю за месяц — это для вкладки «Статистика».

Плитки некликабельны на фазе 1.

## Связь с навигацией

```
Профиль (вкладка)          Аккаунт (push)
     │                          │
     │  кнопка «Аккаунт» ───────┤
     │                          │
     ├─ лекарства, карточки     ├─ обзор + данные аккаунта
     └─ подопечные, шаринг      └─ без дублирования навигации
```

Профиль = **работа с лекарствами и связями**.  
Аккаунт = **идентичность, данные входа, краткий обзор**.

## Открытые вопросы

1. Показывать ли мини-обзор **по всем профилям суммарно** или только по «Мой»?  
   *Рекомендация:* только «Мой» на аккаунте; суммарно — во вкладке «Статистика».

2. Нужна ли кнопка «Удалить аккаунт» (GDPR)?  
   *Пока нет в макете — добавить отдельно, если потребуется.*

3. Показывать ли блок «О приложении» у гостя?  
   *Опционально — только версия, без лишних пунктов.*
